알림 시스템 및 자유 매칭 처리를 위한 REST API Payload Schema

1. DB 원본 응답 데이터 (Before) 자유 매칭 시 null 전달
[
  {
    "requestId": 42,
    "sender": { "id": 12, "name": "황중경" },
    "receiver": { "id": 15, "name": "정성원" },
    "status": "PENDING",
    "targetProjectTitle": null, // 공모전 미지정
    "message": "같이 프로젝트 진행해요!",
    "createdAt": "2026-05-11T15:12:00Z"
  }
]
※ 설명: 특정 프로젝트 공모를 거치지 않고 사용자 간 직접 매칭을 시도할 경우, 데이터베이스의 프로젝트 식별키가 비어 있게 됩니다.

2. 알림 센터 연계용 DTO (After) 결함 보정 및 정렬 추가
{
  "id": "req_42",
  "type": "매칭",
  "title": "황중경님이 팀원 요청을 보냈어요",
  "desc1": "프로젝트 무관 (자유 매칭)", // Null 보정
  "desc2": "\"같이 프로젝트 진행해요!\"",
  "_rawTime": 1778512320000, // 최신순 정렬 보장
  "time": "2026. 5. 11. 오후 3:12:00"
}
※ 해결: 백엔드에서 null 감지 시 한글 치환 처리를 실행하여 화면 렌더링 깨짐을 차단하고, Unix 밀리초 타임스탬프를 더해 최신순 정렬을 보장합니다.
